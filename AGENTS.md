# Agent Notes

## Tooling
- This is a Docusaurus 3.10 site using ESM config (`docusaurus.config.js`) and React 19.
- Use Node >=20; `package.json` declares this and Docusaurus 3 requires modern Node.
- A `package-lock.json` is committed, so prefer `npm ci`/`npm install` and `npm run ...` even though `README.md` still shows `yarn` examples.
- There are no repo scripts for lint, tests, formatting, or typecheck. The main verification command is `npm run build`.

## Commands
- `npm run start` starts the local Docusaurus dev server.
- `npm run build` builds the static site into `build/` and fails on broken links because `onBrokenLinks: 'throw'`.
- `npm run serve` serves the generated `build/` output after a successful build.
- `npm run clear` clears Docusaurus generated state (`.docusaurus`, caches) when routes or generated metadata look stale.
- `npm run deploy` deploys to the `gh-pages` branch; config still uses placeholder `your-org` URLs, so do not treat deployment settings as production-ready without updating them.

## Project Shape
- Root app wiring lives in `docusaurus.config.js`; the docs sidebar is autogenerated from `docs/` via `sidebars.js`.
- Docs pages live in `docs/`; category ordering/labels come from `_category_.json` files and front matter such as `sidebar_position`.
- Blog metadata is centralized in `blog/authors.yml` and `blog/tags.yml`; Docusaurus is configured to warn on unknown inline authors/tags.
- Custom React pages/components are under `src/pages/` and `src/components/`; global Infima/theme overrides are in `src/css/custom.css`.
- Static assets live under `static/` and are referenced from site paths like `/img/...`.

## Gotchas
- `future: { v4: true }` is enabled in Docusaurus config, so avoid patterns that are deprecated for Docusaurus v4.
- `trailingSlash: true` is set; generated URLs should include trailing slashes.
- `build/`, `.docusaurus`, and `.cache-loader` are generated/ignored. Do not edit or commit generated output.
